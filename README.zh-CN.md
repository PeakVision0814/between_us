# Between Us

[English](README.md)

Between Us 是一款以隐私优先为原则、为情侣两个人使用而设计的移动 App。
目标是构建一个用于日常生活、回忆、计划和小型个人工具的共享空间，并且这些
功能只需要真正适合我们自己使用。

## 项目目标

- 构建一个真正的移动 App，而不是网页外壳。
- 让 App 专注于两个人的私密使用场景。
- 从一个小而稳定的情侣 App 基础开始。
- 根据真实的日常需求逐步添加功能。
- 从一开始就考虑数据所有权、隐私和可维护性。

## 计划技术栈

- App：Flutter
- 后端：Supabase
- 数据库：通过 Supabase 使用 PostgreSQL
- 身份认证：Supabase Auth
- 存储：Supabase Storage
- 目标平台：Android 优先，之后支持 iOS

## 第一版原型

第一版原型应该验证基础 App 能力：

- 两个用户可以登录。
- 两个用户可以共享同一个情侣空间。
- 首页展示基础的关系信息。
- 可以创建一条简单的时间线记录或每日留言。
- 可以添加并展示一个基础纪念日。

## 未来模块

- 情侣时间线
- 纪念日
- 每日留言
- 愿望清单
- 共同照片回忆
- 家庭菜单
- 提醒和通知
- 个人资料和情侣设置

## 开发路线

1. 创建 Flutter 项目结构。
2. 定义 App 主题、路由和功能模块布局。
3. 为第一版原型实现仅本地可用的界面。
4. 接入 Supabase 身份认证。
5. 添加情侣空间数据模型和访问规则。
6. 为时间线、留言和纪念日实现同步。
7. 在基础 App 稳定后添加家庭菜单模块。
